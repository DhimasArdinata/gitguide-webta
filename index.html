Panduan Setup & Alur Kerja Git & GitHub

Proyek: Atomic Web TA

Dokumen ini berisi instruksi wajib untuk setup lingkungan dan alur kerja kolaborasi. Ikuti setiap langkah secara berurutan.

BAGIAN 1: SETUP AWAL (HANYA DILAKUKAN SEKALI)

1.1. Membuat & Mengonfigurasi Akun GitHub

  1. Buka https://github.com/signup, buat akun.
  2. Kirim username GitHub yang telah dibuat ke grup WhatsApp untuk diundang ke repository.
  3. Buka email, klik "Accept Invitation" untuk menerima undangan.
  4. Buka https://git-scm.com/downloads, download dan install Git dengan pengaturan default.

1.2. Konfigurasi Git Global

  1. Buka VS Code. Buka Terminal (`Ctrl` + `~`).
  2. Jalankan dua perintah berikut. Ganti placeholder Nama Lengkap dan email.github@example.com dengan data yang sesuai. 
    git config --global user.name "Nama Lengkap"
    git config --global user.email "email.github@example.com"

1.3. Mengunduh Proyek (Clone) - Prosedur Detail

  1. Membuat Folder Kerja Utama (Di Luar VS Code):

    Sebelum membuka VS Code, buat sebuah folder baru di lokasi yang mudah diakses (misalnya di dalam `Documents`). Beri nama folder ini atomic-project. Folder ini akan menjadi wadah untuk proyek kita.

  2. Membuka Folder Kerja di VS Code:

    Buka aplikasi VS Code. Gunakan menu File > Open Folder... dan pilih folder atomic-project yang baru saja dibuat.

  3. Mengunduh Proyek via Terminal VS Code:

    Setelah folder atomic-project terbuka, buka Terminal terintegrasi di VS Code (`Ctrl` + `~`). Terminal akan otomatis aktif di dalam folder tersebut. Jalankan perintah clone:

    git clone https://github.com/DhimasArdinata/atomic-web-ta.git

    Tunggu proses download selesai. Sekarang akan ada folder baru bernama atomic-web-ta di dalam folder atomic-project.

  4. Membuka Folder Proyek Final:

    Ini adalah langkah terakhir dan terpenting. Gunakan menu File > Open Folder... lagi, dan kali ini buka folder atomic-web-ta. Ini adalah folder kerja utama yang akan digunakan seterusnya.

PENTING: Pastikan VS Code sekarang membuka folder atomic-web-ta, bukan folder atomic-project. Semua pekerjaan selanjutnya dilakukan di dalam atomic-web-ta.

BAGIAN 2: SIKLUS KERJA PENGEMBANGAN (WAJIB DIIKUTI SETIAP MEMULAI TUGAS)

2.1. Sinkronisasi Data Terbaru

Selalu mulai dengan data terupdate dari server.

  1. Pindah ke branch main: 
    git checkout main
  2. Tarik perubahan terbaru: 
    git pull origin main

2.2. Membuat Branch Tugas

Buat branch baru dari main. Gunakan username GitHub (huruf kecil) sebagai prefix, diikuti deskripsi tugas yang dipisahkan tanda hubung (-).

  1. Format perintah: git checkout -b username/deskripsi-tugas
  2. Contoh Eksekusi: 
    # Contoh untuk Dhimas
git checkout -b dhimas/integrasi-kontrol-kamera

# Contoh untuk Bagas
git checkout -b bagas/setup-database-sensor

# Contoh untuk Amanda
git checkout -b amanda/desain-halaman-dashboard

# Contoh untuk Audena
git checkout -b audena/perbaikan-bug-login

2.3. Melakukan Pekerjaan (Coding)

Lakukan semua perubahan kode yang diperlukan untuk tugas di dalam branch yang baru dibuat.

2.4. Menyimpan Perubahan (Commit)

Simpan progres pekerjaan.

  1. Siapkan semua file yang berubah untuk disimpan: 
    git add .
  2. Buat commit dengan pesan yang jelas tentang apa yang dilakukan: 
    git commit -m "Menambahkan fungsi streaming video dari kamera"

2.5. Mengunggah Branch (Push)

Unggah branch lokal ke repository di GitHub.

  1. Format perintah: git push origin username/deskripsi-tugas
  2. Contoh Eksekusi: 
    git push origin dhimas/integrasi-kontrol-kamera

2.6. Membuat Pull Request (PR) via Website GitHub

Ajukan perubahan untuk digabungkan ke main. Proses ini wajib dilakukan di website GitHub.

  1. Buka halaman repository: https://github.com/DhimasArdinata/atomic-web-ta.
  2. Klik tombol hijau "Compare & pull request" yang muncul untuk branch yang baru di-push.
  3. Pastikan base: main dan compare: [nama-branch-tugas].
  4. Isi judul dan deskripsi PR secara detail.
  5. Klik "Create pull request".
  6. Tunggu proses code review. Jika ada revisi, lakukan di lokal, lalu commit dan push lagi ke branch yang sama. PR akan otomatis ter-update.
Siklus kerja selesai. Ulangi dari langkah 2.1 untuk tugas berikutnya.
