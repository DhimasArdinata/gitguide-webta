Panduan Setup & Alur Kerja Git & GitHub

Proyek: Atomic Web TA

BAGIAN 1: SETUP AWAL (HANYA DILAKUKAN SEKALI)

1.1. Membuat & Menerima Undangan

  1. Buka https://github.com/signup, buat akun.
  2. Kirim username GitHub ke grup WhatsApp untuk diundang.
  3. Buka email, klik "Accept Invitation".
  4. Buka https://git-scm.com/downloads, download dan install Git dengan pengaturan default.

1.2. Konfigurasi Identitas Git

Perintah ini hanya memberi label nama pada pekerjaan. Ini bukan proses login. Perintah ini tidak menghasilkan output jika berhasil.
  1. Buka VS Code. Buka Terminal (`Ctrl` + `~`).
  2. Jalankan dua perintah berikut (ganti placeholder):

    Input di Terminal:

    $ git config --global user.name "Nama Lengkap"
    $ git config --global user.email "email.github@example.com"

1.3. Menghubungkan VS Code dengan Akun GitHub (Autentikasi)

Ini adalah proses login yang sebenarnya untuk memberi VS Code izin mengakses repository di GitHub. Pilih salah satu metode.

Metode 1: Otorisasi via Browser (Direkomendasikan)

  1. Di VS Code, klik ikon Akun (gambar siluet orang) di pojok kiri bawah.
  2. Pilih "Sign in with GitHub...".
  3. Sebuah browser akan terbuka meminta izin. Klik tombol hijau "Authorize Visual-Studio-Code".
  4. Setelah berhasil, status di ikon Akun akan berubah menunjukkan username GitHub. Proses selesai.

Metode 2: Menggunakan Personal Access Token (PAT) - Alternatif

Gunakan metode ini jika metode pertama gagal.

  1. Langkah A: Membuat Token di Website GitHub
    1. Buka halaman pembuatan token: github.com/settings/tokens/new.
    2. Note: Beri nama deskriptif (misal: `vscode-token-ta`).
    3. Expiration: Pilih durasi (misal: 30 hari).
    4. Scopes: Beri izin. Centang kotak di sebelah `repo`.
    5. Scroll ke bawah dan klik "Generate token".
    2. PENTING: SALIN token yang muncul (contoh: ghp_xxxxxxxx). Token ini hanya ditampilkan sekali. Simpan di tempat yang aman.
  2. Langkah B: Menggunakan Token di Terminal

    Saat melakukan `git clone` atau `git push`, jika terminal meminta Username dan Password:

    1. Username: Masukkan username GitHub.
    2. Password: PASTE TOKEN YANG TELAH DISALIN, bukan password akun GitHub.

1.4. Mengunduh Proyek (Clone) - Prosedur Detail

  1. Membuat Folder Kerja Utama:

    Buat sebuah folder baru di lokasi yang mudah diakses (misal: `Documents`). Beri nama folder ini atomic-project. Folder ini akan menjadi wadah untuk proyek.

  2. Membuka Folder Kerja di VS Code:

    Buka aplikasi VS Code. Gunakan menu File > Open Folder... dan pilih folder atomic-project.

  3. Mengunduh Proyek via Terminal VS Code:

    Buka Terminal di VS Code (`Ctrl` + `~`). Jalankan perintah clone:

    Contoh di Terminal:

    $ git clone https://github.com/DhimasArdinata/atomic-web-ta.git
Cloning into 'atomic-web-ta'...
remote: Enumerating objects: 3, done.
remote: Counting objects: 100% (3/3), done.
remote: Compressing objects: 100% (2/2), done.
remote: Total 3 (delta 0), reused 0 (delta 0), pack-reused 0
Receiving objects: 100% (3/3), done.
  4. Membuka Folder Proyek Final:

    Gunakan menu File > Open Folder... lagi, dan kali ini buka folder atomic-web-ta. Ini adalah folder kerja utama yang akan digunakan seterusnya.

BAGIAN 2: SIKLUS KERJA PENGEMBANGAN

2.1. Sinkronisasi Data Terbaru

Selalu mulai dengan data terupdate dari server.

Contoh di Terminal:

$ git checkout main
Switched to branch 'main'
Your branch is up to date with 'origin/main'.

$ git pull origin main
From https://github.com/DhimasArdinata/atomic-web-ta
 * branch            main       -> FETCH_HEAD
Already up to date.

2.2. Membuat Branch Tugas

Buat branch baru dari `main`. Gunakan username GitHub (huruf kecil) sebagai prefix, diikuti deskripsi tugas yang dipisahkan tanda hubung (-).

Contoh Eksekusi di Terminal:

$ # Contoh untuk Dhimas
$ git checkout -b dhimas/integrasi-kontrol-kamera
Switched to a new branch 'dhimas/integrasi-kontrol-kamera'

$ # Contoh untuk Bagas
$ git checkout -b bagas/setup-database-sensor

$ # Contoh untuk Amanda
$ git checkout -b amanda/desain-halaman-dashboard

$ # Contoh untuk Audena
$ git checkout -b audena/perbaikan-bug-login

2.3. Melakukan Pekerjaan (Coding)

Lakukan semua perubahan kode yang diperlukan untuk tugas di dalam branch yang baru dibuat.

2.4. Menyimpan Perubahan (Commit)

Simpan progres pekerjaan. Perintah `git add .` tidak menghasilkan output jika berhasil.

Contoh di Terminal:

$ git add .

$ git commit -m "Menambahkan fungsi streaming video dari kamera"
[dhimas/integrasi-kontrol-kamera 2abcf13] Menambahkan fungsi streaming video dari kamera
 2 files changed, 45 insertions(+), 3 deletions(-)
 create mode 100644 components/CameraFeed.js

2.6. Membuat Pull Request (PR) via Website GitHub

Ajukan perubahan untuk digabungkan ke `main`. Proses ini wajib dilakukan di website GitHub.

  1. Buka halaman repository: https://github.com/DhimasArdinata/atomic-web-ta.
  2. Klik tombol hijau "Compare & pull request" yang muncul untuk branch yang baru di-push.
  3. Pastikan base: main dan `compare: [nama-branch-tugas]`.
  4. Isi judul dan deskripsi PR secara detail.
  5. Klik "Create pull request".
  6. Tunggu proses code review. Jika ada revisi, lakukan di lokal, lalu commit dan `push` lagi ke branch yang sama. PR akan otomatis ter-update.
Siklus kerja selesai. Ulangi dari langkah 2.1 untuk tugas berikutnya.
